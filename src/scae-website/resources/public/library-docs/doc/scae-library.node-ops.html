scae-library.node-ops

This namespace contains helper functions to help make dealing
with the Abstract Syntax Tree when designing style rules easier

entry-type-check?

(entry-type-check? ast-entry key)
Used to create ast-entry checks

get-in-nested-entry

(get-in-nested-entry ast-entry path-pairs)
Gets the nested entry of the node following the path-pair. The path-pair
is a vector of vectors. The inner vectors have 2 values, n and child-name, which
are passed into the get-nth-name-child function.

get-node-children

(get-node-children ast-entry)
Get the children of a node

get-node-name

(get-node-name ast-entry)
Get the nameof a node

get-nth-child

(get-nth-child ast-node n)
Get the nth child of the ast node. Uses zero indexing (0 is first position).
Ignores symbol table function calls and line-numbers

get-nth-named-child

(get-nth-named-child ast-node n child-name)
Retrieve the nth instance of the target child. Uses zero indexing

get-parsed-node-result

(get-parsed-node-result ast-node)
This returns the parsed node result. If the parsed node result is a string,
it reads the string and returns the eval'd result. This is due to elsewhere
in the program, the style analyser, having to quote the parsed node result so
that internal hash-maps would not try to be evaluated as a function if they
were at the head of a list and eval were called on them

node-contains-nested-child?

(node-contains-nested-child? ast-node child-name)
Check if a node contains a certain child. Note that child-name is the name
of the node / token and not its value.

node-contains-nested-token?

(node-contains-nested-token? ast-entry token-key token-value)
Returns true if the node contains a token that has a value of token-value
with key of token-key anywhere in its children or their sub-trees

node-contains-top-layer-child?

(node-contains-top-layer-child? ast-node child-name)
Check if a node contains a certain direct child. Note that child-name is the name
of the node / token and not its value.

node?

(node? ast-entry)
Check if the current ast-entry is of type node

st-func-calls?

(st-func-calls? ast-entry)
Check if the current ast-entry is of type st-func-calls

sub-node-contains-nested-child?

(sub-node-contains-nested-child? ast-node sub-node-name child-name)
Check if a sub node contains a certain child. Note that child-name is the name
of the node / token and not its value.

sub-node-contains-top-layer-child?

(sub-node-contains-top-layer-child? ast-node sub-node-name child-name)
Check if a sub node contains a certain direct child. Note that child-name is the name
of the node / token and not its value.

token?

(token? ast-entry)
Check if the current ast-entry is of type token
